### Continuous Delivery (CD) support model

{{site.data.reuse.long_name}} releases 10.x and later use the continuous delivery (CD) support model, except for versions 10.2.x which are [Extended Update Support (EUS) releases](#extended-update-support-eus-releases).

Product defect fixes and security updates are only available in the latest CD version, in line with the IBM's continuous delivery lifecycle policy (for more information, see the {{site.data.reuse.short_name}} [support lifecycle addendum](https://www.ibm.com/support/pages/node/6589953){:target="_blank"}. Ensure you stay current with the installation of the latest CD update packages.

### Extended Update Support (EUS) releases

{{site.data.reuse.long_name}} versions 10.2.x, running only on Red Hat OpenShift 4.6, benefit from defect support until 27 October 2022, in line with the Extended Update Support provided for Red Hat OpenShift 4.6.

**Important:** To qualify for Extended Update Support (EUS), ensure you have the supported OpenShift version, and only use the listed {{site.data.reuse.short_name}} operator versions for the operand. For more information about operator requirements for EUS releases, see the [Cloud Pak documentation](https://www.ibm.com/docs/en/cloud-paks/cp-integration/2020.4?topic=installing-operators#extended-update-support-eus){:target="_blank"}. No fixes will be provided for Extended Update Support (EUS) releases after 27 October 2022. This does not affect the End of Support (EoS) date, or the support for moving to later versions for customers that have entitlement.

## Support policy

{{site.data.reuse.long_name}} incorporates both IBM proprietary and open source components, including Apache Kafka. In case of a problem with any of these components, IBM will investigate, identify, and provide a fix when possible. Where the fix applies to an open source component, IBM will work with the open source community to contribute the fix to the open source project as appropriate.

If you encounter client-side issues when using {{site.data.reuse.short_name}} with clients that are not provided by IBM, IBM can assist you in working with the open source community to resolve those issues.
